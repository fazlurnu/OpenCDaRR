# future-features

Deferred features and known gaps: real, considered, and explicitly **not** built yet. One file
per feature — what it is, why it's deferred, and the **target version** (`docs/roadmap.md`'s
scheme) it's earmarked for, so it doesn't get silently forgotten *or* silently smuggled in early.

Not a wishlist: only things that came up as a real design question during actual work land here
(each file says where it came from). A version marker here is a placeholder, not a commitment —
confirm/adjust when the roadmap is actually revisited.
